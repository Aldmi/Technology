
1. FROM
FROM <image> [AS <name>]
При желании можно добавить имя к новому этапу сборки, добавив AS nameв FROM инструкцию.
Имя может быть использовано в последующем FROM и COPY --from=<name|index> инструкции для ссылки на image, построенное на этом этапе.

2. RUN
RUN <command>
RUN ["executable", "param1", "param2"]
RUN Инструкция будет выполнять любые команды в новом слое поверх текущего изображения и фиксации результатов.
Полученное зафиксированное image будет использовано для следующего шага в Dockerfile.

3. COPY
COPY [--chown=<user>:<group>] <src>... <dest>
COPY [--chown=<user>:<group>] ["<src>",... "<dest>"] - Для путей содержащих пробелы
Эта --chown функция поддерживается только в файлах Docker, используемых для создания контейнеров Linux, и не будет работать в контейнерах Windows. 

<src>  Можно указать несколько ресурсов, но пути к файлам и каталогам будут задваться относительно источника контекста сборки.
<dest> Абсолютный путь, или путь относительно WORKDIR, в котором источник будет скопирован внутри контейнера назначения.

COPY test relativeDir/   # adds "test" to `WORKDIR`/relativeDir/
COPY test /absoluteDir/  # adds "test" to /absoluteDir/

При желании COPYпринимает флаг, --from=<name|index>который можно использовать для установки исходного местоположения на предыдущий этап сборки (созданный с помощью FROM .. AS <name>),
который будет использоваться вместо контекста сборки, отправленного пользователем. Флаг также принимает числовой индекс, назначенный для всех предыдущих этапов сборки,
начатых с FROMинструкцией. Если этап сборки с указанным именем не может быть найден, вместо него делается попытка использовать изображение с тем же именем.

COPY подчиняется следующим правилам:

   *** <src> Путь должен быть внутри контекста билда; Вы не можете этого сделать COPY ../something /something, потому что первым шагом docker buildявляется отправка каталога контекста (и его подкаталогов) демону Docker.
   *** Если <src>это каталог, копируется все содержимое каталога, включая метаданные файловой системы.
   
4. EXPOSE  - проброс портов
   EXPOSE 80
   Вы можете указать множество инструкций EXPOSE для указания, какие порты должны быть открыты. Также инструкция EXPOSE полезна для проброса портов между контейнерами.
